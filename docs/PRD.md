# PRD: HoldRect

> 按住修饰键拖动鼠标，画一个彩虹条纹强调框，松手消失。

---

## 1. Summary

HoldRect 是一个极致轻量的桌面常驻工具：按住修饰键 + 鼠标左键拖动，在任何界面上画出彩虹条纹矩形框，松开鼠标后框自动消失。专为录屏和演示场景设计——不需要时完全隐形，需要时一秒激活。用 Rust 开发，跨平台 (Windows / macOS / Linux)，目标常驻内存 < 5MB。

## 3. Background

**问题**: 录屏创作者和演示者在强调屏幕区域时，现有工具要么太重 (Epic Pen ~50MB)，要么需要切模式 (ZoomIt 要按 Ctrl+2)，要么只支持单一平台。没有一个工具能做"按一下就画框、松手就没了"这种零摩擦的交互。

**为什么现在做**:
- Rust 桌面 GUI 生态成熟 (winit, wgpu)
- Windows 上的透明全屏 overlay 技术路径清晰 (WS_EX_LAYERED + WS_EX_TRANSPARENT)
- GitHub 上同类轻量工具有明确的 star 获取模式 (GIF 演示 + 一句话说清)

**为什么值得做**: 竞品调研显示市面上没有"橡皮筋选择框 + 彩虹条纹 + 松手消失"的工具。这是一个差异化明确的细分切入点。

## 7. Solution

### 7.1 UX / 用户流程

```
[常驻后台, 无界面]
        │
        ▼
用户按住修饰键 (Ctrl/Alt/Win 可配置)
        │
        ▼
用户按住鼠标左键 → 在按住位置出现框的起点
        │
        ▼
用户拖动鼠标 → 彩虹条纹矩形框实时跟随绘制
        │
        ▼
        │  拖拽中可按数字键 toggle:
        │  1 → Pinned (冻结), 2 → Spotlight (聚光灯)
        │  状态弹窗从屏幕顶部滑入显示
        │
        ▼
用户松开鼠标左键
        │
        ├─ Transient 模式 (默认): 框立即消失
        │
        └─ Pinned 模式: 框冻结在屏幕上，按 Esc 清除所有
        │
        ▼
用户松开修饰键 → 回到常驻后台状态
```

**修饰键行为规范**:
- 修饰键必须先于鼠标按下才触发（防止正常操作误触发）
- 修饰键可在配置文件中自定义
- 默认修饰键: `Alt` (不容易与其他软件冲突, Ctrl会被很多快捷键占用)

**框的视觉效果**:
- 边框: 彩虹渐变色，沿周长流动 (流动动画)
- 边框宽度: 4px (可配置)
- 填充: 无填充 / 半透明浅色 (可配置)
- 动画: 彩虹色沿矩形周长循环流动，流速约 60px/s

#### 7.1.1 交互模型

**核心手势（永远不变）**:

`修饰键 + 鼠标左键拖拽` = 画框。无论当前处于什么模式，这个手势的含义不变。

**行为修饰符 — 数字键 toggle**:

在拖拽过程中（或拖拽开始前），按下数字键可以 toggle 额外行为。数字键是 per-rect 的 — 每次画新框时状态重置为默认。

| 按键 | Toggle 的行为 | 图标 | 再按一次 |
|------|-------------|------|---------|
| `1` | Pinned（松手后框冻结在屏幕上） | 📌 | 关闭 Pinned |
| `2` | Spotlight（框外区域变暗，聚光灯效果） | 🔦 | 关闭 Spotlight |

数字键可叠加：按 `1` 后再按 `2` = Pinned + Spotlight。再按 `1` = 取消 Pinned，只剩 Spotlight。

两种触发路径，结果相同：
- **预按**：`修饰键 + 1 + 拖拽` — 先按数字键再拖拽
- **中途按**：`修饰键 + 拖拽中 + 按 1` — 拖拽过程中随时 toggle

也支持同时按：`修饰键 + 1 + 2 + 拖拽` — 一步到位。

**状态系统**:

- **Per-rect 状态**：每个框独立决定自己的行为。画新框时，数字键状态重置为 transient（默认）。用户按 `1` 只影响当前正在画的这一个框。
- **全局默认模式**：通过托盘菜单或配置文件设置，影响所有新框的初始行为。适合长期偏好（如"我默认就要 pinned"）。
- 优先级：per-rect toggle > 全局默认模式。

**状态提示弹窗**:

当用户 toggle 数字键时，在屏幕顶部弹出状态提示。

| 属性 | 定义 |
|------|------|
| 位置 | 当前画框所在屏幕的正上方居中（多显示器时，始终出现在鼠标光标所在的屏幕） |
| 触发 | 用户按下数字键 toggle 的瞬间 |
| 内容 | 当前所有激活状态的图标 + 文字。例如 `📌 Pinned`、`🔦 Spotlight`、`📌 Pinned · 🔦 Spotlight` |
| 动效 | 从顶部滑入 → 停留约 0.5s → 滑出消失 |
| 实时更新 | 用户连续 toggle 时，弹窗内容实时更新，0.5s 计时器重置。例如按 `1` 显示 `📌 Pinned`，0.3s 内再按 `2` 弹窗立刻变为 `📌 Pinned · 🔦 Spotlight` 并重新开始 0.5s 倒计时，直到最后一次 toggle 后 0.5s 才滑出消失 |
| 空状态 | 所有 toggle 都关闭时（回到 transient），弹窗显示 `Transient` 或不弹出 |

**快捷键说明书弹窗**:

按住 `` 修饰键 + ` ``（反引号键）不放，弹出快捷键速查卡。松开按键即消失。

| 属性 | 定义 |
|------|------|
| 行为 | Hold to show — 按住显示，松手消失 |
| 位置 | 当前屏幕居中 |
| 内容 | 当前所有可用快捷键的列表 |
| 与数字键的区别 | 数字键是 toggle（按一下切换），说明书是 hold（按住才显示）— 两种交互模式不混淆 |

**Esc 键**:

- 有已冻结的框时：按下 Esc 清除所有已冻结（pinned）的框
- 正在拖拽绘制时：按下 Esc 取消当前绘制（框消失，回到空闲状态）

**托盘菜单**:

托盘菜单提供全局默认模式的切换入口，影响所有新框的初始行为。

```
  HoldRect
  ─────────────────
  默认模式:
  ○ Transient (松手消失)
  ● Pinned (松手冻结)
  ○ Spotlight (聚光灯)
  ○ Pinned + Spotlight
  ─────────────────
  清除所有标注 (Esc)
  ─────────────────
  设置 / 退出
```

**配置文件 `~/.holdrect/config.toml`**:

```toml
[general]
modifier = "Alt"                  # 修饰键: Alt / Ctrl / Shift / Win
default_mode = "transient"        # 默认模式: transient / pinned / spotlight / pinned_spotlight
border_width = 4                  # 边框宽度 (px)
color = "rainbow"                 # 颜色模式: rainbow / 十六进制色值 (如 "#ff0000")
```

`default_mode` 对应托盘菜单中的全局默认模式，影响所有新框的初始行为。per-rect 的数字键 toggle 始终可以临时覆盖此设置。

### 7.3 Technology

| 层 | 选型 | 理由 |
|---|------|------|
| 语言 | Rust | 零运行时、低内存、跨平台 |

### 竞品对比

| 工具 | 平台 | 价格 | 开源 | 内存占用 | 核心特点 | HoldRect 优势 |
|------|------|------|------|---------|---------|-------------|
| **Epic Pen** | Windows | $25-30 Pro | ✗ | ~20-50MB | 画笔、高亮、形状、文本、截图、白板 | 开源、免费、轻 10 倍、零模式切换 |
| **ZoomIt** | Windows | 免费 (Microsoft) | ✗ | ~10-15MB | 屏幕缩放、绘图、倒计时、白板 | 开源、跨平台、彩虹动画、Spotlight 聚光灯、鼠标穿透 |
| **gInk** | Windows | 免费 | ✓ (MIT) | ~15-30MB | 画笔、高亮、橡皮擦、箭头、截图 | 轻 3-5 倍、无模式切换、彩虹动画、跨平台 |
| **Gromit-MPX** | Linux (X11) | 免费 | ✓ (GPL) | ~5-10MB | 自由绘图、箭头、矩形 | 跨平台、聚光灯、动画效果、Wayland 支持 |
| **ScreenBrush** | Win/Mac | 免费/付费 | ✗ | ~10-20MB | 高亮、绘图、聚光灯、自动消失 | 开源、更轻、Linux 支持 |
| **Fluor** | macOS 14+ | 免费 | ✓ | ~85MB | 自由线条、箭头、矩形、聚光灯 | 轻 17 倍、跨平台 |

**HoldRect 差异化定位**:

1. **彩虹流动动画** — 零竞品提供渐变动画边框，全部用纯色。这是视觉辨识度的核心。
2. **修饰键+拖拽的零切换交互** — 竞品需要切模式（点托盘图标、按快捷键进入绘图模式）。HoldRect 按住修饰键直接画，松手消失，零摩擦。
3. **WS_EX_TRANSPARENT 鼠标穿透** — 竞品的 overlay 会拦截鼠标事件。HoldRect 的框是纯视觉强调，不干扰底层应用操作。
4. **Per-rect 数字键 toggle** — 拖拽过程中按数字键即时切换 Pinned/Spotlight，实时弹窗反馈。竞品无此交互模式。
5. **极低内存** — Rust 原生二进制，目标 <5MB。最轻的竞品也要 10MB+。
6. **真正的跨平台** — 原生 Win32 API + macOS + Linux，不是 Electron 包装。

**一句话定位**: Epic Pen 的易用性 + ZoomIt 的轻量 + 独一无二的彩虹动画和零模式切换 — 开源、免费、2MB。
