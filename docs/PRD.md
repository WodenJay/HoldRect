# PRD: HoldRect

> 按住修饰键拖动鼠标，画一个彩虹条纹强调框，松手消失。

---

## 1. Summary

HoldRect 是一个极致轻量的桌面常驻工具：按住修饰键 + 鼠标左键拖动，在任何界面上画出彩虹条纹矩形框，松开鼠标后框自动消失。专为录屏和演示场景设计——不需要时完全隐形，需要时一秒激活。用 Rust 开发，跨平台 (Windows / macOS / Linux)，目标常驻内存 < 5MB。

## 3. Background

**问题**: 录屏创作者和演示者在强调屏幕区域时，现有工具要么太重 (Epic Pen ~50MB)，要么需要切模式 (ZoomIt 要按 Ctrl+2)，要么只支持单一平台。没有一个工具能做"按一下就画框、松手就没了"这种零摩擦的交互。

**为什么现在做**:
- Rust 桌面 GUI 生态成熟 (winit, wgpu, softbuffer)
- Windows 上的透明全屏 overlay 技术路径清晰 (WS_EX_LAYERED + WS_EX_TRANSPARENT)
- GitHub 上同类轻量工具有明确的 star 获取模式 (GIF 演示 + 一句话说清)

**为什么值得做**: 竞品调研显示市面上没有"橡皮筋选择框 + 彩虹条纹 + 松手消失"的工具。这是一个差异化明确的细分切入点。

## 7. Solution

### 7.1 UX / 用户流程

```
[常驻后台, 无界面]
        │
        ▼
用户按住修饰键 (Ctrl/Alt/Win 可配置)
        │
        ▼
用户按住鼠标左键 → 在按住位置出现框的起点
        │
        ▼
用户拖动鼠标 → 彩虹条纹矩形框实时跟随绘制
        │
        ▼
用户松开鼠标左键 → 框立即消失
        │
        ▼
用户松开修饰键 → 回到常驻后台状态
```

**修饰键行为规范**:
- 修饰键必须先于鼠标按下才触发（防止正常操作误触发）
- 修饰键可在配置文件中自定义
- 默认修饰键: `Ctrl` (最不容易与其他软件冲突)

**框的视觉效果**:
- 边框: 彩虹渐变色，沿周长流动 (流动动画)
- 边框宽度: 4px (可配置)
- 填充: 无填充 / 半透明浅色 (可配置)
- 动画: 彩虹色沿矩形周长循环流动，流速约 60px/s

### 7.3 Technology

| 层 | 选型 | 理由 |
|---|------|------|
| 语言 | Rust | 零运行时、低内存、跨平台 |
| 窗口创建 | `windows` crate (Win32 API) | 透明 overlay 需要精细控制窗口样式 |
| 渲染 | `softbuffer` + CPU 光栅化 | 极低依赖，矩形绘制不需要 GPU |
| 全局键鼠监听 | `rdev` crate | 跨平台全局输入事件监听 |
| 配置 | `toml` + `serde` | Rust 生态标准方案 |
| 系统托盘 | `tray-icon` crate | 跨平台托盘图标 |
| 打包 | `cargo-dist` 或手动 NSIS | 跨平台发布 |

**技术路径关键决策**:

1. **为什么用 softbuffer 而不是 wgpu?** 矩形绘制是 2D 操作，softbuffer 是 CPU 光栅化，依赖少、编译快、内存低。wgpu 的 GPU 管线对画一个矩形来说杀鸡用牛刀。

2. **为什么不用 Tauri?** Tauri 带 WebView，内存开销大 (20MB+)，与 <5MB 目标冲突。

3. **全局输入监听的安全性**: `rdev` 需要 OS 级别的输入钩子权限。Windows 上需要处理 UAC 提升窗口的问题（无法在提权窗口上画框）。

### 7.4 Assumptions

| # | Assumption | 验证方式 |
|---|-----------|---------|
| 1 | `rdev` 在 Windows 上能可靠监听修饰键 | v0.1 spike 验证 |
| 2 | softbuffer 能实现 60fps 透明 overlay | v0.1 性能测试 |
| 3 | WS_EX_TRANSPARENT 能让鼠标事件穿透 overlay | v0.1 手动验证 |
| 4 | 彩虹条纹在 1080p 录屏压缩后仍然清晰 | v0.2 录屏验证 |

| 6 | Linux Wayland 用户量小，layer-shell 不支持可以接受 | 社区反馈 |
