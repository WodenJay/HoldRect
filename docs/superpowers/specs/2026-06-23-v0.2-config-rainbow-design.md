# v0.2 Design: Config + Rainbow Border

> Incremental iteration on HoldRect. Adds TOML config file and rainbow gradient border. Skips auto-launch (registry) for later.

---

## 1. Scope

**In scope:**
- TOML config file (`~/.holdrect/config.toml`)
- Configurable modifier key (Alt/Ctrl/Shift/Win)
- Configurable border width (1-20px)
- Configurable border color: solid hex or rainbow mode
- Rainbow HSV gradient border with flowing animation

**Out of scope (deferred to later iterations):**
- Auto-launch via registry (v0.2b or v0.4)
- Hot-reload config (v0.4)
- Fill effects (v0.4)
- Rounded corners (v0.4)

---

## 2. Config System

### 2.1 TOML Format

```toml
# ~/.holdrect/config.toml
# All fields optional. Missing fields use defaults.

modifier = "Alt"       # Alt | Ctrl | Shift | Win
border_width = 4       # 1-20
color = "#FF0000"      # hex color string OR "rainbow"
```

### 2.2 Rust Struct

```rust
// src/config.rs

pub struct AppConfig {
    pub modifier_vk_codes: Vec<u32>,
    pub border_width: i32,
    pub color_mode: ColorMode,
}

pub enum ColorMode {
    Solid { r: u8, g: u8, b: u8 },
    Rainbow,
}

impl AppConfig {
    /// Load config from ~/.holdrect/config.toml. Infallible — returns defaults on any failure.
    pub fn load() -> Self { ... }
}
```

### 2.3 Loading Behavior

- Path: `dirs::home_dir()` / `.holdrect/config.toml`
- `AppConfig::load()` called once in `main()` before anything else
- `dirs::home_dir()` returns `None` → print warning to stderr, use all defaults
- Missing file → use all defaults, no error
- Missing field → use default for that field
- Malformed TOML → print warning to stderr, use all defaults
- Invalid modifier string → print warning, default to "Alt"
- Invalid color string → print warning, default to "#FF0000"
- border_width clamped to 1-20

### 2.4 Defaults

| Field | Default |
|-------|---------|
| modifier | "Alt" |
| border_width | 4 |
| color | "#FF0000" (solid red) |

### 2.5 Modifier → VK Code Mapping

| Config value | VK codes |
|---|---|
| "Alt" | VK_MENU (0x12), VK_LMENU (0xA4), VK_RMENU (0xA5) |
| "Ctrl" | VK_CONTROL (0x11), VK_LCONTROL (0xA2), VK_RCONTROL (0xA3) |
| "Shift" | VK_SHIFT (0x10), VK_LSHIFT (0xA0), VK_RSHIFT (0xA1) |
| "Win" | VK_LWIN (0x5B), VK_RWIN (0x5C) |

### 2.6 Color Parsing

- `"rainbow"` → `ColorMode::Rainbow`
- `"#RRGGBB"` → `ColorMode::Solid { r, g, b }` (hex parse)
- Anything else → fallback to `ColorMode::Solid { 255, 0, 0 }`

### 2.7 New Dependencies

| Crate | Purpose |
|-------|---------|
| `toml` | TOML parsing |
| `serde` (with `derive`) | Deserialization |
| `dirs` | Home directory path |

## 3. Hook Integration

### 3.1 Changes

`start_hook_listener` signature changes:

```rust
// Before:
pub fn start_hook_listener(tx: Sender<InputEvent>, proxy: EventLoopProxy<()>)

// After:
pub fn start_hook_listener(tx: Sender<InputEvent>, proxy: EventLoopProxy<()>, modifier_codes: Vec<u32>)
```

### 3.2 Storage

New static in `hook.rs`:

```rust
static MODIFIER_CODES: OnceLock<Vec<u32>> = OnceLock::new();
```

Initialized once in `start_hook_listener`.

### 3.3 decide_keyboard Change

```rust
// Before:
fn decide_keyboard(vk_code: u32, is_key_down: bool) -> Option<InputEvent> {
    let is_alt = vk_code == VK_MENU.0 as u32
        || vk_code == VK_LMENU.0 as u32
        || vk_code == VK_RMENU.0 as u32;
    if !is_alt { return None; }
    Some(InputEvent::ModifierChanged { pressed: is_key_down })
}

// After:
fn decide_keyboard(vk_code: u32, is_key_down: bool, modifier_codes: &[u32]) -> Option<InputEvent> {
    if !modifier_codes.contains(&vk_code) {
        return None;
    }
    Some(InputEvent::ModifierChanged { pressed: is_key_down })
}
```

The hook proc calls it with the stored codes:
```rust
let codes = MODIFIER_CODES.get().expect("MODIFIER_CODES not set");
if let Some(event) = decide_keyboard(kb.vkCode, is_key_down, codes) {
```

### 3.4 What Stays Same

- `SHOULD_SUPPRESS`, `DRAG_IN_PROGRESS` atomics
- `decide_mouse` function logic — unchanged
- Mouse hook proc — unchanged
- `call_next_hook_ex` calls — unchanged

### 3.5 Naming Cleanup

Current `decide_mouse` has parameter `ctrl_held` but the caller passes `alt_held`. Since modifier is now configurable, rename to `modifier_held`:

```rust
// Before:
fn decide_mouse(msg: u32, pt: (i32, i32), should_suppress: bool, drag_in_progress: bool, ctrl_held: bool)

// After:
fn decide_mouse(msg: u32, pt: (i32, i32), should_suppress: bool, drag_in_progress: bool, modifier_held: bool)
```

Also rename the local variable in `mouse_hook_proc` from `alt_held` to `modifier_held` for consistency.

## 4. Rainbow Rendering

### 4.1 HSV Math (new functions in overlay.rs)

```rust
/// Convert HSV (h: 0-360, s: 0-1, v: 0-1) to RGB
fn hsv_to_rgb(h: f32, s: f32, v: f32) -> (u8, u8, u8)

/// Position along rectangle perimeter as fraction 0.0..1.0 (clockwise from top-left)
/// Top edge: 0.0..0.25, Right edge: 0.25..0.5, Bottom edge: 0.5..0.75, Left edge: 0.75..1.0
/// Precondition: x0 < x1 && y0 < y1 (caller guarantees via normalize_rect + x1>x0 guard in draw_border)
fn perimeter_position(x: i32, y: i32, x0: i32, y0: i32, x1: i32, y1: i32) -> f32
```

Both are pure functions, no Win32 calls, fully unit-testable.

### 4.2 Animation Timing

- `let time_secs = std::time::SystemTime::now().duration_since(UNIX_EPOCH).as_secs_f32();`
- `let time_offset = time_secs * FLOW_SPEED;`
- `const FLOW_SPEED: f32 = 0.1;` — one full rainbow cycle every ~10 seconds
- Wall-clock anchored: every drag continues from current time position (no reset per drag). Intentional — consistent with "flowing stripe" visual.
- No stored animation state in `App`.

### 4.3 draw_border Changes

The `set_red_pixel` closure becomes `set_pixel(x, y, r, g, b)`.

For `ColorMode::Solid { r, g, b }`: all border pixels get the same color. Same as current behavior but configurable color.

For `ColorMode::Rainbow`:
```
for each border pixel (x, y):
    pos = perimeter_position(x, y, x0, y0, x1, y1)
    hue = ((pos + time_offset) % 1.0) * 360.0
    (r, g, b) = hsv_to_rgb(hue, 1.0, 1.0)
    set_pixel(x, y, r, g, b)
```

### 4.4 Function Signature

```rust
// Before:
fn draw_border(window: &Window, start: (i32, i32), current: (i32, i32))

// After:
fn draw_border(window: &Window, start: (i32, i32), current: (i32, i32), border_width: i32, color_mode: &ColorMode)
```

No `DrawConfig` wrapper — pass `border_width` and `color_mode` directly. Avoids a redundant struct that must stay in sync with `AppConfig`.

### 4.5 App Struct Change

```rust
pub struct App {
    window: Option<Window>,
    state: AppState,
    input_rx: Receiver<InputEvent>,
    border_width: i32,        // new
    color_mode: ColorMode,    // new
}
```

`App::new` takes `border_width: i32` and `color_mode: ColorMode` parameters.

## 5. Data Flow

```
main()
  ├── AppConfig::load()          → config
  ├── start_hook_listener(tx, proxy, config.modifier_vk_codes)
  ├── start_tray(exit_tx)
  └── run_overlay(event_loop, input_rx, config.border_width, config.color_mode)
        └── App { border_width, color_mode }
              └── draw_border(window, start, current, self.border_width, &self.color_mode)
```

Updated `run_overlay` signature:
```rust
pub fn run_overlay(event_loop: EventLoop<()>, input_rx: Receiver<InputEvent>, border_width: i32, color_mode: ColorMode)
```

## 9. Risks

| Risk | Mitigation |
|------|-----------|
| `dirs` crate adds binary size | ~20KB, negligible for a desktop app |
| HSV math per-pixel is slow | ~10 ops/pixel, 15K pixels = 150K flops. GDI overhead dominates. |
| Config file race (edit while running) | No hot-reload this iteration. Load once at startup. |
| Win key modifier might trigger Start menu | Same suppression approach as Alt. VK_LWIN/VK_RWIN in hook should work. If not, document as known limitation. |
| Shift-as-modifier suppresses typing | SHOULD_SUPPRESS is set on modifier keydown. With Shift configured, all keyboard input while Shift is held gets suppressed via the hook. Users must be aware. Document in README. |
