# HoldRect v0.1 Design Spec

> 最小可用版本: Windows 上实现核心交互, 验证技术路径.

---

## 1. Overview

v0.1 实现 HoldRect 的核心交互循环: 按住 Ctrl + 鼠标左键拖拽画矩形框, 松手消失. 目标是验证技术路径可行, 后续版本在此基础上加彩虹效果和配置.

**验收标准**:
- `cargo build --release` 编译通过
- 运行后按住 Ctrl + 鼠标拖拽出现矩形框
- 松开鼠标框消失
- 不影响其他应用的正常鼠标操作
- 常驻内存 < 10MB

**约束**:
- v0.1 仅支持单显示器, 主显示器
- v0.1 仅支持 Windows
- DPI: 设置为 System DPI Aware, 保证像素数学确定性

---

## 2. Tech Stack

| 组件 | 选型 | 用途 |
|------|------|------|
| 窗口管理 | winit 0.30+ | 跨平台窗口创建, 事件循环 (PRD原定纯Win32, 改用winit简化跨平台) |
| 渲染 | softbuffer | CPU 光栅化, 画矩形框 |
| 全局输入监听 | rdev | 跨平台键鼠事件 |
| 系统托盘 | tray-icon | 托盘图标 + 右键退出 |
| 通道 | std::sync::mpsc | 线程间通信 |
| Windows API | windows crate | WS_EX_TRANSPARENT 等窗口样式 |

**错误处理策略**: v0.1 所有不可恢复错误 (窗口创建失败等) 直接 panic + 错误信息.

## 3. Architecture

### 三线程 + 通道通信

```
┌─────────────┐    mpsc::channel    ┌──────────────────┐
│  rdev 线程   │ ──InputEvent──────> │                  │
│ (input.rs)  │                     │   主线程          │
└─────────────┘                     │ (winit event loop)│
                                    │                  │
┌─────────────┐    mpsc::channel    │  state.rs 状态机  │
│  tray 线程   │ ──AppExit────────> │  overlay.rs 渲染  │
│ (tray.rs)   │                     │                  │
└─────────────┘                     └──────────────────┘
```

- 主线程: winit event loop + softbuffer 渲染, 消费两个 channel
- rdev 线程: 监听全局键鼠, 发送 `InputEvent`
- tray 线程: 托盘图标, 发送 `AppExit` 信号 (tray-icon 内部处理消息泵, 无需手动 GetMessage)

**假设**: rdev `listen()` 内部处理 Windows 消息泵 (WH_KEYBOARD_LL/WH_MOUSE_LL 要求). v0.1 spike 验证.

**PRD 偏差说明**: PRD 原定纯 Win32 API 创建窗口, 实际改用 winit + softbuffer 简化跨平台路径, windows crate 仅用于 WS_EX_TRANSPARENT 和 DPI 设置.

## 4. Module Structure

```
src/
├── main.rs          # 入口: 启动线程, 初始化, winit event loop
├── state.rs         # AppState + InputEvent + 状态转换 (纯函数, 可测试)

└── tray.rs          # 系统托盘
```

日志: v0.1 不引入 logging 框架, 后续版本按需加.

## 5. State Machine

### AppState

```rust
#[derive(Debug, Clone, PartialEq)]
enum DrawingState {
    Idle,
    Armed,
    Drawing { start: (i32, i32), current: (i32, i32) },
}

struct AppState {
    state: DrawingState,
}
```

### States

- **Idle**: 后台待命, overlay 隐藏, 零 CPU
- **Armed**: Ctrl 按住, 等待鼠标动作, overlay 隐藏
- **Drawing**: 鼠标拖拽中, overlay 可见, 实时渲染矩形. 记录 start_pos 和 current_pos

### Transitions

```
Idle ──[ModifierDown]──> Armed ──[MouseDown]──> Drawing
  ^                       |                       |
  └──[ModifierUp]─────────┘                       |
  └──[ModifierUp]─────────────────────────────────┘

Drawing ──[MouseUp]──> Armed (overlay 隐藏, 可再次拖拽)
```

行为说明:
- Drawing + MouseUp → Armed: 立即隐藏 overlay (框消失), 但 Ctrl 仍按住, 用户可再次点击拖拽画新框 (PRD 未明确此行为, 是合理增强)
- Armed + ModifierUp → Idle: Ctrl 松开, 完全回到待命状态
- Drawing + ModifierUp → Idle: Ctrl 松开, 取消绘制, 隐藏 overlay

### InputEvent enum

```rust
enum InputEvent {
    ModifierChanged { pressed: bool },
    MouseButtonDown { x: i32, y: i32 },
    MouseButtonUp { x: i32, y: i32 },
    MouseMove { x: i32, y: i32 },
}
```

`MouseButtonUp` 带坐标, 防止 MouseMove 在快速释放时丢失最终位置.

### process_event() — 纯函数

```rust
fn process_event(state: &AppState, event: &InputEvent) -> AppState
```

无副作用, 可直接单元测试.

## 6. Window + Rendering

### 窗口创建

- winit `Window::with_attributes()`:
  - `transparent: true`
  - `decorations: false`
  - `always_on_top: true`
  - 全屏尺寸 (主显示器物理像素分辨率)
- DPI: 在 `main()` 最开头调用 `SetProcessDpiAwarenessContext(DPI_AWARENESS_CONTEXT_SYSTEM_AWARE)`, 必须在任何窗口创建之前
- Windows 平台: 通过 `raw-window-handle` 获取 HWND, 用 `SetWindowLongPtrW` 设置 `WS_EX_TRANSPARENT | WS_EX_LAYERED` 实现鼠标穿透
- winit 0.30+ 使用 `ApplicationHandler` trait

### 渲染

- softbuffer `Surface` 绑定窗口
- 像素格式: BGRA (Windows 原生), `0x00000000` = 全透明
- 每帧: 清空为全透明
- 矩形边框: 亮红 `0xFFFF0000` (u32: A=0xFF, R=0xFF, G=0x00, B=0x00), 4px 宽
- 仅在 Drawing 状态渲染, Armed/Idle 时 `set_visible(false)` 隐藏窗口
- 隐藏时额外调用 Win32 `ShowWindow(SW_HIDE)` 防止窗口出现在 Alt+Tab
- 已知限制: v0.1 不做抗锯齿, 矩形边缘有锯齿

### 帧率

- `ControlFlow::WaitUntil` ~60fps (16ms 间隔)
- Drawing 时每帧重绘, Idle/Armed 时 Wait (零 CPU)

## 7. Input Handling

- rdev `listen()` 在独立线程, 阻塞运行
- 回调只做: 事件 → `InputEvent` → `tx.send()`
- 监听事件:
  - `Key::ControlLeft` / `Key::ControlRight` → `ModifierChanged`
  - 鼠标左键按下/松开 → `MouseButtonDown` / `MouseButtonUp`
  - 鼠标移动 → `MouseMove { x, y }`
- 忽略: 右键、中键、其他键 (v0.1 不处理)

---

## 8. System Tray

- `tray-icon` 创建托盘图标 (内嵌 .ico 字节数组, 无外部文件)
- 右键菜单: "退出 HoldRect"
- tray-icon 内部处理 Windows 消息泵, 无需手动 GetMessage 循环
- 退出 → 发送 `AppExit` → 主线程 `event_loop.exit()`

## 10. Testing

| 层 | 方法 | 覆盖 |
|----|------|------|
| state.rs | 单元测试 | 所有状态转换路径 |
| input.rs | 不测 (rdev 需要真实输入) | — |
| overlay.rs | 手动 `cargo run` | 窗口创建, 渲染, 鼠标穿透 |
| 全流程 | 手动验收 | Ctrl+拖拽→框→松手→消失 |

### 状态机测试用例

1. Idle + ModifierDown → Armed
2. Armed + MouseDown(pos) → Drawing { start: pos, current: pos }
3. Drawing + MouseMove(pos) → Drawing { current: pos }
4. Drawing + MouseUp(pos) → Armed
5. Armed + ModifierUp → Idle
6. Drawing + ModifierUp → Idle (取消绘制)
7. Idle + MouseDown → Idle (不响应)
8. Idle + MouseMove → Idle (不响应)
9. Idle + ModifierUp → Idle (无操作)
10. Armed + MouseMove → Armed (无操作, 只有 MouseDown 触发 Drawing)
11. Drawing + MouseDown → Drawing (忽略, 不更新坐标)
12. Drawing + ModifierDown → Drawing (忽略, 已经在绘制中)
